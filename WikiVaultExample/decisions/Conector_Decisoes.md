# Conector: Decisões Arquiteturais

Este documento centraliza todas as decisões de arquitetura e design tomadas no **grapy-wiki**.

- [[flask-sobre-fastapi]] — Por que usamos Flask em vez de FastAPI para o SSR.
- [[arquitetura-ddd]] — Adoção de Domain-Driven Design para separar a lógica de negócio da web.
- [[migracao-django]] — Como e por que mantemos suporte planejado a uma migração para Django.
- [[zero-npm]] — A regra estrita de proibir NPM e frameworks JS para garantir máxima segurança.
- [[motor-fisica-customizado]] — O uso de Canvas e Vanilla JS puro ao invés de D3.js.
- [[wikilinks-vs-frontmatter]] — A escolha de links `[[...]]` no lugar de YAML para montar o grafo.
- [[rbac-por-pastas]] — Como a estrutura de pastas do Obsidian resolve segurança de notas e anexos.
- [[grapy-wiki-vs-obsidian]] — Por que criar uma engine própria ao invés de usar o Obsidian Publish.
- [[nome-grapy-wiki]] — A origem do nome (Grafo, Python, Wiki).
